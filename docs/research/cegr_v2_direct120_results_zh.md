# CEGR V2 Direct120 实验结果

## 1. 实验结论

CEGR V2 Direct120 已完成 2 步冒烟、120 步正式训练和固定 700 题配对评测。主指标没有改善：历史 Search-R1 baseline 的平均 Exact Match（EM）为 0.2371，CEGR V2 为 0.2257，差值为 -0.0114。配对 bootstrap 95% 置信区间为 [-0.0400, 0.0157]，精确 McNemar 检验 p=0.4926。因此，本实验不能声称 CEGR V2 对主指标有效。

CEGR V2 同时呈现了明确的效率变化：平均回答长度下降 14.93%，平均搜索次数下降 0.0371，证据覆盖率上升 0.0043。多跳集合的 EM 上升 0.0050，但单跳集合的 EM 下降 0.0333。结果支持“更短、更保守的搜索轨迹”这一行为变化，却不支持“总体问答准确率提高”。

## 2. 实验配置

| 项目 | 配置 |
| --- | --- |
| GPU | 1 x NVIDIA A800 80GB |
| 基座模型 | Qwen2.5-1.5B-Instruct |
| 强化学习算法 | GRPO |
| 检索器 | CPU BM25，Top-3 |
| 最大搜索轮数 | 4 |
| 正式训练更新数 | 120 |
| train batch size | 32 |
| 每题采样数 | 5 |
| 学习率 | 1e-6 |
| warmup ratio | 0.95 |
| 随机种子 | 42 |
| 训练耗时 | 15320 秒（4 小时 15 分 20 秒） |
| 评测集 | 7 个数据集，每个 100 题，共 700 题 |

正式训练从原始 Qwen2.5-1.5B-Instruct 开始，没有从 2 步冒烟 checkpoint 续训。训练在第 40、80、120 步保存 checkpoint，最终 checkpoint 完整存在。120 步训练期间未观察到 OOM、NaN 或未处理异常。

## 3. 可复核流程

1. 运行离线预检，核验 V1 冻结副本、数据互斥、Ray worker 共置、BM25 服务和测试集。
2. 运行 2 步冒烟，确认检索、生成、奖励、参数更新和 checkpoint 保存全部可用。
3. 从原始模型启动 EFF120 正式训练。
4. 对 baseline 与 EFF120 使用相同的 700 题、数据 SHA、seed 和 engine seed 进行独立推理。
5. 逐题对齐两份 JSONL，计算 EM、token-F1、检索行为、配对 bootstrap 置信区间和精确 McNemar 检验。

冒烟实验耗时 391 秒。其两个 batch 的 informative fallback 比例分别为 0.5714 和 0.5000，且 mixed-group reward mismatch 为 0，说明分组和奖励实现按预期工作。

## 4. 总体结果

| 指标 | Baseline | CEGR V2 | 差值 |
| --- | ---: | ---: | ---: |
| EM | 0.2371 | 0.2257 | -0.0114 |
| Token-F1 | 0.2970 | 0.2893 | -0.0077 |
| 证据覆盖率 | 0.3000 | 0.3043 | +0.0043 |
| 平均回答 token 数 | 759.03 | 645.74 | -113.29 |
| 平均搜索次数 | 1.1600 | 1.1229 | -0.0371 |
| 平均重复搜索次数 | 0.0671 | 0.0629 | -0.0043 |
| 平均无效搜索次数 | 0.0014 | 0.0014 | 0.0000 |

逐题转移中，baseline 错而 V2 对的题有 48 道，baseline 对而 V2 错的题有 56 道。总体 EM 差值的 95% 置信区间跨过 0，不能排除抽样波动；但点估计方向为负，也未达到预注册的至少 +0.02 EM 改善标准。

## 5. 分数据集结果

| 数据集 | Baseline EM | CEGR V2 EM | 差值 |
| --- | ---: | ---: | ---: |
| NQ | 0.24 | 0.23 | -0.01 |
| TriviaQA | 0.51 | 0.46 | -0.05 |
| PopQA | 0.36 | 0.32 | -0.04 |
| HotpotQA | 0.19 | 0.23 | +0.04 |
| 2WikiMultihopQA | 0.25 | 0.21 | -0.04 |
| MuSiQue | 0.03 | 0.02 | -0.01 |
| Bamboogle | 0.08 | 0.11 | +0.03 |

按任务类型聚合后，多跳题从 0.1375 上升到 0.1425，差值为 +0.0050，95% 置信区间为 [-0.0325, 0.0425]；单跳题从 0.3700 下降到 0.3367，差值为 -0.0333，95% 置信区间为 [-0.0767, 0.0133]。两者都未达到统计显著，但单跳下降是总体负差值的主要来源。

## 6. 结果解释

EFF fallback 在一个采样组的 EM 全为零、但候选答案仍有部分 token 重叠时提供稠密信号。理论上，它可缓解纯 EM 奖励在困难题上的全零组问题。冒烟和训练日志证明该信号确实被触发，且分组边界正确，因此本次负结果不能简单归因于“改进代码没有生效”。

更合理的解释是奖励目标与最终指标存在偏差。Token-F1 能奖励部分重叠，但不保证最终短答案完全正确；配合长度和检索行为的变化，策略学会了生成更短的轨迹，却没有把节省的 token 稳定转化为更高 EM。HotpotQA 的改善说明这种信号对部分需要组合证据的问题可能有价值；TriviaQA、PopQA 等单跳集合的下降则说明统一施加该信号会干扰已有的直接作答能力。

因此，下一版不应继续简单增加训练步数或提高 EFF 权重。更有依据的方向是任务或状态自适应门控：仅在多跳特征明显、检索证据充分且答案重叠可由证据支持时启用 fallback；单跳或高置信直接作答样本继续使用原始 EM 奖励。另一个低成本诊断是先在与最终 700 题互斥的 140 题 pilot 上比较第 40、80、120 步 checkpoint，判断单跳退化是否来自过度训练，再决定是否评测更早 checkpoint。

## 7. 因果边界与报告口径

本对比估计的是“分组修正 + EFF”组合方法相对于复用历史 baseline 的差异。它没有单独识别 F1 fallback 与分组修正各自的因果贡献，也不是同一时刻新训练的随机双臂实验。由于训练条件和最终评测协议已固定，该结果可作为课程实验中的有效负结果和后续改进依据，但不能包装成成功提升。

建议在报告中使用以下表述：

> CEGR V2 显著改变了推理效率，使平均响应长度减少 14.93%，但没有提高主要准确率指标。700 题配对评测中 EM 下降 1.14 个百分点，差异不显著。分组分析显示多跳任务略有改善、单跳任务退化，提示后续应采用按任务难度或证据状态门控的奖励，而非对所有样本统一应用稠密 fallback。
