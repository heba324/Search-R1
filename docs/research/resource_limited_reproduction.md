# Search-R1 v5 资源受限复现：方法口径与科学边界

核查日期：2026-07-12

## 结论

课程实验以第一篇主论文 [Search-R1: Training LLMs to Reason and Leverage Search Engines with Reinforcement Learning](https://arxiv.org/abs/2503.09516) 的 **COLM 2025 / arXiv v5** 为理论对象。工程实现继续使用 `codex/course-reproduction` 分支，并要求当前提交继承自课程基线 `eaa5a66c0ed779d195a4bd0e165f0c73b99f12de`。

准确命名是：

> **Search-R1 v5 核心机制的资源受限复现与改进**

在改进实验尚未执行时，阶段性结果应写成“Search-R1 v5 核心机制的资源受限复现”，不能声称严格数值复现，也不能声称已经完成改进。

## 1. 不要混合三个版本对象

| 对象 | 在本项目中的用途 |
| --- | --- |
| 第一篇论文 arXiv v5 / COLM 2025 | 理论方法、公式和论文结果的主要引用来源 |
| 第一篇论文 v1 | 只用于简述版本演进，不单独重跑 |
| 官方仓库 v0.3 / 第二篇经验研究 | 奖励、模型、检索器和稳定性改进的参考，不定义基础复现 |

官方仓库后来加入格式奖励、新模型、新检索配置和第二篇经验研究的实验脚本。直接把仓库最新能力都称为“原始 Search-R1”会导致版本口径失真。

## 2. 本次保留的 Search-R1 科学内核

本项目必须真实运行以下闭环：

```text
模型生成 <search>查询</search>
  -> 环境调用检索器并返回 Top-3
  -> 文档写入 <information>...</information>
  -> 模型继续推理，最多交互 4 轮
  -> 输出 <answer>答案</answer>
  -> 最终答案 Exact Match 结果奖励
  -> GRPO 更新策略
```

`actor_rollout_ref.actor.state_masking=true` 必须保留，使环境注入的检索文档 token 不参与策略梯度。课程基线不加入格式奖励或中间过程奖励；否则它不再是单纯的 GRPO + EM 基线。

## 3. 课程配置与论文 v5 的差异

| 维度 | 论文 v5 代表性设置 | 课程设置 | 结论限制 |
| --- | --- | --- | --- |
| 模型 | Qwen2.5 3B/7B 等 | Qwen2.5-1.5B-Instruct | 容量与初始能力不同 |
| RL | PPO、GRPO 均研究 | GRPO | 属于论文方法，但优化动态不同 |
| 检索 | Wikipedia 2018 + E5 稠密检索 | 同语料 + CPU BM25 | 文档分布发生变化 |
| 训练数据 | NQ + HotpotQA | NQ + HotpotQA | 保持一致 |
| 检索返回 | Top-3 | Top-3 | 保持一致 |
| 最大轮次 | 4 | 4 | 保持一致 |
| 奖励 | 最终答案结果奖励 | 归一化 EM | 保持核心思想 |
| 硬件 | 大规模多卡 | 1×A800 80GB | batch、吞吐与轨迹规模不同 |
| 评测 | 七数据集完整设置 | 每个数据集固定 100 题 | 只能做项目内部配对比较 |

因此，论文表格绝对值不能作为课程实验“成功/失败”的直接阈值。可靠结论来自相同资源协议下的配对比较。

## 4. 实验矩阵

- M0：原始 Qwen2.5-1.5B-Instruct，不做 RL。
- M1：课程 Search-R1 基线，GRPO + 最终答案 EM。
- M2：一个独立改进，例如动态搜索成本或重复查询惩罚。

M0 和 M1 已有可执行的 Pre-RL/Post-RL 评测流程。M2 在真正实现和运行前只属于计划。后续消融应一次只改变一个变量，并固定模型 revision、训练与评测数据、BM25 revision、seed、Top-3 和最大轮数。

## 5. 训练前后对比证据

只比较 EM 不足以说明模型“更会搜索”。课程评测同时记录固定评测子集整体的：

- 平均搜索次数；
- 平均有效与无效搜索次数；
- 平均重复查询次数；
- 使用搜索的样本比例；
- 平均响应 token 数。

`evaluate_pre_rl.sh` 与 `evaluate.sh` 使用同一份固定七数据集子集。Post-RL 评测会生成 `pre-post-comparison.json`，其中包含逐数据集 EM 差值和评测子集整体的搜索行为差值。这样可以区分“答案变好”“搜索更多/更少”和“重复或无效搜索变化”。

## 6. 后续改进依据

第一篇论文 v5 暴露了结果奖励稀疏、重复/无效搜索、固定 Top-k 和训练稳定性等问题。第二篇 [经验研究](https://arxiv.org/abs/2505.15117) 可用于选择奖励、模型初始化或检索器方面的单因素改进，但不能倒过来定义基础复现版本。

优先考虑的小改进包括动态搜索成本、重复查询惩罚、自适应 Top-k、检索相关性辅助奖励或搜索停止策略。正式报告必须明确区分 M1 基线与 M2 改进，并给出消融证据。

## 7. 推荐报告表述

> 本实验以 Search-R1 的 COLM 2025/arXiv v5 版本为理论复现对象。考虑到原论文模型规模、稠密索引和多卡训练成本较高，本实验使用 Qwen2.5-1.5B-Instruct、GRPO、CPU BM25 和单张 A800 80GB，在保留多轮推理—搜索、Top-3、最大四轮交互、检索 token 掩码及最终答案 EM 奖励的前提下开展资源受限复现。实验目标是验证核心训练闭环以及训练前后的性能与搜索行为趋势，而非逐项复现论文表格中的绝对数值。
